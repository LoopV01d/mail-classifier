# Mail Classifier

Учебный проект для хакатона по предмету «Технологии программирования».

Проект читает письма из папки `inbox`, определяет категорию каждого письма по ключевым словам и копирует письма в соответствующие папки внутри `output`.

## Что делает программа

Программа выполняет простую автоматическую сортировку писем.

Порядок работы:

1. Берёт все файлы из папки `inbox`.
2. Пропускает скрытые файлы, например `.DS_Store`.
3. Читает каждое письмо.
4. Определяет тему, отправителя и текст письма.
5. Классифицирует письмо по ключевым словам.
6. Копирует письмо в папку нужной категории внутри `output`.
7. Выводит в терминал итоговую статистику по категориям.
8. Если файл невозможно прочитать, программа не падает, а выводит причину ошибки.

Важно: программа именно копирует письма в `output`, а не удаляет их из `inbox`. Поэтому исходные файлы остаются на месте.

## Категории писем

В проекте используются следующие категории:

- `spam` — рекламные и подозрительные письма, розыгрыши, выигрыши, фишинг.
- `critical` — срочные технические проблемы, аварии, сбои, недоступность сервисов.
- `access` — запросы на доступы, права, VPN, GitLab, Active Directory и другие системы.
- `hardware` — проблемы с техникой: ноутбук, монитор, гарнитура, клавиатура, принтер.
- `software` — проблемы с программами: 1С, Zoom, Excel, Chrome, Outlook, лицензии и ошибки запуска.
- `documents` — счета, акты, договоры, закрывающие документы, бухгалтерские документы.
- `tasks` — рабочие поручения, просьбы подготовить отчёт, таблицу или данные.
- `info` — информационные уведомления, плановые работы, дайджесты, встречи и напоминания.
- `unknown` — письма, которые не подошли ни под одну категорию.

## Основные файлы

### `src/mail_message.py`

В этом файле находится класс `MailMessage`.

Он хранит данные одного письма:

- имя файла;
- тему письма;
- отправителя;
- текст письма.

Также в классе есть метод `get_full_mail_text()`, который объединяет тему и текст письма и переводит их в нижний регистр. Это нужно, чтобы классификатору было проще искать ключевые слова.

### `src/mail_reader.py`

В этом файле находится класс `MailReader`.

Он отвечает за чтение файлов из папки `inbox`.

Класс умеет:

- читать обычные текстовые файлы;
- читать некоторые `.json` файлы;
- пробовать разные кодировки;
- пропускать бинарные файлы;
- возвращать ошибку, если файл пустой, слишком большой или не подходит для чтения.

### `src/mail_classification.py`

В этом файле находится класс `MailClassifier`.

Он хранит словари ключевых слов и определяет категорию письма.

Сначала программа проверяет тему письма по отдельному словарю `subject_dict`. Потом проверяет весь текст письма по словарю `category_dict`.

Если ни одно ключевое слово не найдено, возвращается категория `unknown`.

### `src/main.py`

Главный файл программы.

Он:

1. Создаёт объект `MailReader`.
2. Создаёт объект `MailClassifier`.
3. Читает все письма из `inbox`.
4. Классифицирует письма.
5. Создаёт папки категорий внутри `output`.
6. Копирует письма в нужные папки.
7. Печатает итоговую статистику.

### `run.sh`

Bash-скрипт для удобного запуска проекта.

Он:

- проверяет, что папка `inbox` существует;
- считает количество писем;
- создаёт папку `output`;
- запускает `src/main.py`;
- сохраняет лог запуска в файл `run.log`.

## Как запустить проект

Откройте папку проекта в VS Code.

Затем откройте терминал и выполните команду:

```bash
bash run.sh
```

Также можно запустить программу напрямую:

```bash
python3 src/main.py
```

На Windows может использоваться команда:

```bash
py src/main.py
```

После запуска в папке `output` появятся папки с категориями писем.

## Как запустить тесты

В проекте есть тесты для классификатора.

Для запуска тестов выполните:

```bash
python3 -m pytest
```

На Windows:

```bash
py -m pytest
```

Если `pytest` не установлен, установите его командой:

```bash
python3 -m pip install pytest
```

На Windows:

```bash
py -m pip install pytest
```

После успешного запуска тестов в терминале должно появиться сообщение, что тесты пройдены.

## Что проверяют тесты

Файл `tests/test_classifier.py` проверяет, что классификатор правильно относит письма к категориям:

- `spam`;
- `critical`;
- `access`;
- `hardware`;
- `software`;
- `documents`;
- `tasks`;
- `info`;
- `unknown`.

Также есть проверки:

- письма без ключевых слов попадают в `unknown`;
- классификатор работает с разным регистром букв;
- если письмо содержит признаки нескольких категорий, срабатывает приоритетная категория.

## Как работает классификация

Классификация основана на поиске ключевых слов.

Примеры:

- если в письме есть слова `выигрыш`, `розыгрыш`, `банковская карта`, письмо относится к `spam`;
- если есть слова `срочно`, `авария`, `работа остановлена`, письмо относится к `critical`;
- если есть слова `доступ`, `vpn`, `gitlab`, письмо относится к `access`;
- если есть слова `ноутбук`, `монитор`, `принтер`, письмо относится к `hardware`;
- если есть слова `1с`, `лицензия`, `outlook`, письмо относится к `software`;
- если есть слова `счёт`, `акт`, `договор`, письмо относится к `documents`;
- если есть слова `подготовить`, `задача`, `поручение`, письмо относится к `tasks`;
- если есть слова `плановые работы`, `дайджест`, `напоминание`, письмо относится к `info`.

## Почему выбран такой подход

Мы выбрали классификацию по ключевым словам, потому что проект учебный и должен быть сделан на основе тем, которые изучались на курсе:

- базовый Python;
- классы и объекты;
- работа с файлами;
- модули;
- обработка исключений;
- тестирование;
- Bash;
- Git.

Такое решение простое, понятное и его можно объяснить на защите.
