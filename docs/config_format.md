# Формат файла конфигурации категорий

Файл `config/categories.json` управляет тем, по каким правилам система раскладывает письма по папкам. Чтобы добавить, убрать или переименовать категорию — **достаточно отредактировать этот файл, код менять не нужно**.

## Структура

```json
{
  "default_category": "unknown",
  "case_sensitive": false,
  "categories": [
    {
      "name": "имя_категории",
      "priority": 1,
      "description": "Зачем эта категория",
      "keywords_subject": ["слова в теме"],
      "keywords_body": ["слова в теле письма"],
      "regex_patterns": ["шаблоны на регулярных выражениях"]
    }
  ]
}
```

## Поля верхнего уровня

| Поле | Назначение |
|---|---|
| `default_category` | Имя папки, куда уходят письма, не подошедшие ни под одно правило |
| `case_sensitive` | Учитывать ли регистр при поиске. Обычно `false` — чтобы «Срочно» и «срочно» считались одинаково |
| `categories` | Список правил классификации |

## Поля категории

| Поле | Обязательное | Назначение |
|---|---|---|
| `name` | да | Имя категории (станет именем выходной папки) |
| `priority` | да | Порядок проверки. **Меньшее число = выше приоритет.** Категория `critical` с `priority: 1` будет проверяться раньше `software` с `priority: 5` |
| `description` | нет | Комментарий — зачем нужна категория. Не влияет на работу |
| `keywords_subject` | хотя бы одно из трёх | Слова, которые ищутся в теме письма |
| `keywords_body` | хотя бы одно из трёх | Слова, которые ищутся в теле письма |
| `regex_patterns` | хотя бы одно из трёх | Регулярные выражения для сложных случаев. Ищутся во всём тексте письма (тема + тело) |

## Как добавить новую категорию

Пример: добавляем категорию «отпуск» для писем про заявления на отпуск.

1. Открыть `config/categories.json`
2. В массив `categories` дописать:

```json
{
  "name": "vacation",
  "priority": 9,
  "description": "Заявления на отпуск и больничный",
  "keywords_subject": ["отпуск", "больничный"],
  "keywords_body": [
    "прошу согласовать отпуск",
    "ежегодный отпуск",
    "больничный лист",
    "период нетрудоспособности"
  ],
  "regex_patterns": []
}
```

3. Запустить программу:

```bash
bash run.sh --config config/categories.json
```

В `output/` появится папка `vacation/` с подходящими письмами.

## Как изменить приоритет

Например, мы хотим, чтобы спам проверялся **раньше** критичных писем (чтобы фишинг с словом «срочно» не попадал в critical):

1. В категории `spam` поменять `priority` с `2` на `0`
2. Запустить программу заново

## Если конфиг неверный

Программа **не запустится**, но напишет понятную причину, например:

- `Категория 'critical' не имеет обязательного поля 'priority'`
- `Категория 'spam' встречается дважды`
- `Категория 'access', regex '[unclosed' невалиден`
- `Некорректный JSON в config/categories.json: ...`

Это значит, что в файле опечатка — исправить и запустить снова.

## Регулярные выражения (для продвинутых)

Регулярные выражения позволяют ловить шаблоны, а не отдельные слова. Несколько полезных:

| Шаблон | Что ловит |
|---|---|
| `\\[CRITICAL\\]` | Метку `[CRITICAL]` в теме |
| `ERR_\\d{3}` | Коды ошибок типа `ERR_500`, `ERR_762` |
| `#\\d{4,}` | Номера заявок типа `#12345` |
| `(счёт|счет)\\s*№\\s*\\d+` | Номера счетов |

**Важно:** в JSON слэш нужно экранировать дважды: `\\d` вместо `\d`.
